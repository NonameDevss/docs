## Prerequisite

Before calling background endpoints, complete [Authentication and token flow](/en/guides/authentication).

## Supported endpoints

- `POST /api/v1/background/remove/url`
- `POST /api/v1/background/remove/base64`
- `POST /api/v1/background/remove/multipart`

All three endpoints require:

- Header `Authorization: <managed_token>`

## Example: remove by URL

```bash
curl -X POST 'https://api.cutbg.net/api/v1/background/remove/url' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: <managed_token>' \
  -d '{
    "image_url": "https://example.com/photo.jpg"
  }'
```

Example response:

```json
{
  "image_base64": "iVBORw0KGgoAAAANSUhEUg..."
}
```

## Decode `image_base64` in memory

Use the `image_base64` value from the API response and decode it to an in-memory image object.

    ```python
    import base64
    from io import BytesIO
    from PIL import Image

    image_base64 = response_json["image_base64"]
    image_bytes = base64.b64decode(image_base64)
    image = Image.open(BytesIO(image_bytes))  # PIL image in memory
    ```

    ```java
    import java.util.Base64;
    import java.io.ByteArrayInputStream;
    import java.awt.image.BufferedImage;

    String imageBase64 = responseJson.get("image_base64").asText();
    byte[] imageBytes = Base64.getDecoder().decode(imageBase64);
    ```
