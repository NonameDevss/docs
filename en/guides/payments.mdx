## Subscription payment flow

1. Read plans: `GET /api/v1/payments/options`.
2. Create payment link: `POST /api/v1/payments/link`.
3. Check invoice list: `GET /api/v1/payments/invoices`.
4. Check invoice details: `GET /api/v1/payments/invoices/{invoice_id}`.

All endpoints except `GET /payments/options` require:

- Header `token: <jwt_token>`

## Example: create payment link

```bash
curl -X POST 'http://localhost:8000/api/v1/payments/link' \
  -H 'Content-Type: application/json' \
  -H 'token: <jwt_token>' \
  -d '{
    "plan": "month"
  }'
```
