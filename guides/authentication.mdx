## Required order

Auth endpoints:

- `POST /api/v1/auth/password`
- `POST /api/v1/auth/device`
- `POST /api/v1/auth/firebase`
- `POST /api/v1/auth/webapp`

Then use this order:

1. Authenticate with one of the auth endpoints.
2. Get `token` from the auth response.
3. Send `token: <jwt_token>` for protected endpoints.
4. Issue a managed token before calling background endpoints.

## Step 1: authenticate and get JWT token

```bash
curl -X POST 'http://localhost:8000/api/v1/auth/password' \
  -H 'Content-Type: application/json' \
  -d '{
    "email": "you@example.com",
    "password": "your-password"
  }'
```

Example response:

```json
{
  "id": 1,
  "email": "you@example.com",
  "language_code": "en",
  "token": "<jwt_token>"
}
```

## Step 2: use JWT token in `token` header

```bash
curl -X GET 'http://localhost:8000/api/v1/user/me' \
  -H 'token: <jwt_token>'
```

## Step 3: issue managed token for background endpoints

```bash
curl -X POST 'http://localhost:8000/api/v1/token/issue' \
  -H 'token: <jwt_token>'
```

Response:

```json
{
  "token": "<managed_token>",
  "expires": "2027-02-26T12:00:00"
}
```

Use this token in `Authorization` for `/background/remove/*` endpoints.

<Note>
Do not add `Bearer` prefix. Pass only the managed token value.
</Note>

## Обязательный порядок

Auth-эндпоинты:

- `POST /api/v1/auth/password`
- `POST /api/v1/auth/device`
- `POST /api/v1/auth/firebase`
- `POST /api/v1/auth/webapp`

Дальше используйте такой порядок:

1. Пройдите авторизацию через один из auth-эндпоинтов.
2. Получите `token` из ответа auth.
3. Передавайте `token: <jwt_token>` для защищенных эндпоинтов.
4. Перед background-эндпоинтами выпустите managed-токен.

## Шаг 1: авторизация и получение JWT-токена

```bash
curl -X POST 'http://localhost:8000/api/v1/auth/password' \
  -H 'Content-Type: application/json' \
  -d '{
    "email": "you@example.com",
    "password": "your-password"
  }'
```

Пример ответа:

```json
{
  "id": 1,
  "email": "you@example.com",
  "language_code": "en",
  "token": "<jwt_token>"
}
```

## Шаг 2: используйте JWT-токен в заголовке `token`

```bash
curl -X GET 'http://localhost:8000/api/v1/user/me' \
  -H 'token: <jwt_token>'
```

## Шаг 3: выпустите managed-токен для background-эндпоинтов

```bash
curl -X POST 'http://localhost:8000/api/v1/token/issue' \
  -H 'token: <jwt_token>'
```

Ответ:

```json
{
  "token": "<managed_token>",
  "expires": "2027-02-26T12:00:00"
}
```

Используйте этот токен в `Authorization` для `/background/remove/*`.

<Note>
Префикс `Bearer` не нужен. Передавайте только значение managed-токена.
</Note>
